# 瀚海AI审查系统 — 项目开发经验总结

## 一、项目概述

- **技术栈**：Electron + React + TypeScript + Vite + Tailwind CSS + SQLite
- **核心功能**：AI 审查任务管理、文件分析、历史记录

## 二、开发历程关键节点

### 1. UI/UX 重设计（参考 aura.build）

- **深空渐变背景**：`--space-deepest: #030712`，营造深邃太空感
- **玻璃拟态（Glass Morphism）**：半透明 + backdrop-blur 实现毛玻璃效果，配合辉光（glow）增强层次感
- **霓虹配色系统**：蓝-紫-品红渐变（neon blue / purple / magenta），统一视觉语言
- **AmbientOrbs 动态光晕层**：大尺寸模糊圆形元素，随鼠标或定时缓慢移动，增加氛围感
- **Canvas 粒子场特效（ParticleField）**：基于 Canvas 2D 的粒子系统，渲染星空/粒子连线效果

### 2. 性能优化

- **CSS 动画复杂度降低**：移除 framer-motion 依赖，改用轻量 CSS `anim-up` 类实现入场动画，减少 JS 动画开销
- **engine.ts 提前 emit 与 AnimatePresence 竞态修复**：确保状态更新在动画完成后再提交，避免 UI 撕裂
- **JSON.stringify 快照去重**：在状态变更前做深度比较，相同快照跳过渲染，减少无效 re-render
- **Electron 启动白屏修复**：将 `isDev` 判断逻辑改为 `!app.isPackaged`，确保生产模式正确加载 `file://` 协议

### 3. 关键 Bug 修复

| Bug 描述 | 根因 | 修复方案 |
|---|---|---|
| 新建审查须点两次才能进入 | `TaskNew.tsx` 的 `useEffect` 在组件挂载时无条件执行 `nav()` 跳转 | 引入 `didSubmit` ref，仅在用户实际提交后才触发跳转 |
| 首次点击路由全空白 | React `StrictMode` 双挂载 + `useEffect` 中 `nav()` 污染 HashRouter 内部状态 | 将导航逻辑移至事件回调中触发，避免挂载阶段副作用 |
| Sidebar `layoutId` 消失 | framer-motion 的 `layoutId` 属性在路由切换时丢失引用 | 移除 `layoutId`，改用纯 CSS transition 处理布局动画 |

### 4. 桌面端部署踩坑

- **asar 打包内容**：`electron-main.cjs`、`preload.cjs`、`dist/`、`node_modules/` 必须正确纳入 asar 包，缺一不可
- **node_modules 嵌套结构**：`sql.js` 必须保持 `node_modules/sql.js/` 层级（而非扁平化放到 `node_modules/` 根），否则 `require('sql.js')` 找不到模块
- **isPackaged 判断失效**：部分系统环境下 `app.isPackaged` 返回 `false`，需硬编码 `isDev = false` 强制走 `file://` 协议加载
- **Canvas z-index 遮挡问题**：`position: fixed` 的粒子 Canvas 需确保 `z-index` 低于主体内容层，否则会遮挡所有交互元素

## 三、技术架构经验

- Electron 生产模式从 `app.asar` 内加载 `file://` 路径的静态资源
- Vite 构建产物为带哈希名的 JS/CSS 文件，`index.html` 自动引用，无需手动管理
- **asar 部署流程**：`vite build` → 提取 asar → 替换 dist 产物 → 重新打包 asar → 替换 `.app` 内 asar → 重启应用
- 开发迭代建议优先用 `vite dev server` 热更新调试，确认无误后再走打包部署流程

## 七、技术债务与改进建议

### 1. 构建部署流程正规化
- **现状**：手动 vite build → 提取 asar → 替换文件 → npx asar pack → 替换 .app 内 asar
- **建议**：使用项目中已配置的 electron-builder（`npm run electron:build`），一键完成构建 + 打包 + 签名，避免手动 asar 操作带来的 node_modules 结构错误和 isPackaged 判断问题

### 2. 粒子场性能优化
- **现状**：100 个粒子 + requestAnimationFrame 全量重绘，在低配设备可能掉帧
- **建议**：
  - 引入 requestAnimationFrame 帧率节流（目标 30fps 而非 60fps）
  - 粒子数量根据设备性能自适应（检测帧率动态调整 PARTICLE_COUNT）
  - 连线检测当前为 O(n²) 复杂度，可考虑空间哈希网格加速

### 3. 死代码清理
- TopBar.tsx：未被 App.tsx 引用，为死代码
- sdk/components.tsx（ReviewInput/Monitor/Config）：未被当前路由引用
- electron-main_\*.cjs 旧备份：已挪至 backup/ 目录

### 4. 路由组件残留问题
- TaskMonitor 空状态中使用 `<Link to="/new">` 跳转，当路由结构调整时需同步更新

### 5. 开发工作流建议
- 日常开发优先使用 `npm run electron:dev`（Vite dev server + Electron），热更新秒级生效
- 确认功能无误后再走 electron-builder 打包，避免反复手动 asar 操作
- 桌面端验证时注意区分 dev 模式（读 localhost:5188）和生产模式（读 app.asar）

### 6. 窗口配置固化
- 窗口尺寸：1440×920（最小 1024×680）
- 已移除配置：titleBarStyle: 'hiddenInset'、trafficLightPosition、vibrancy、backdropFilter、glass/blur CSS

### 7. SQLite 持久化架构
- 通过 useDB.ts hook 桥接 Electron IPC → SQLite
- 数据库路径：`app.getPath('userData')/hanhai.db`
- 表结构：configs、tasks、outputs、reports
