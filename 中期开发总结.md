## 1. Engine Core: Dynamic Debate Round Count

### 1.1 Problem

The system always defaulted to 3 debate rounds, unable to adapt dynamically based on the complexity of the review content. The round judge's configured phase was incorrectly assigned to `debate_summarizer`, and its system_prompt was an older version ("judge based on debate situation" — but it was actually invoked during the preparation phase, resulting in a context mismatch).

| Stage | Approach | Issue |

### 1.3 当前算法

- **内容长度**（每 800 字 +1，上限 5）

综合分 → 轮数映射（1-10 轮全量覆盖）。

## 3. Build & Deployment Fixes

### 3.1 应用崩溃（isDev 硬编码）

**根因**：`electron-main.cjs` 中 `const isDev = !app.isPackaged` 在打包后仍返回 `true`，Electron 尝试连接 `http://localhost:5188/` 失败导致白屏。

**修复**：`isDev` 硬编码为 `false`。

### 3.1 Application Crash (isDev Hardcoding)

**Root cause**: `const isDev = !app.isPackaged` in `electron-main.cjs` still returned `true` after packaging; Electron attempted to connect to `http://localhost:5188/` and failed, resulting in a white screen.

**Fix**: Hardcoded `isDev` to `false`.

### 3.2 tsconfig 路径错误

`config/tsconfig.app.json` 中 `include: ["src"]` 指向不存在的 `config/src/`，应指向 `../src`。修复后 `npm run build` 正常通过。

### 3.2 tsconfig Path Error
