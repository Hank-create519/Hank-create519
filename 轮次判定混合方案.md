# 轮次判定混合方案（方案 C）

## 当前状态（v1.0）

已部署纯本地规则引擎 `estimateRoundCount()`，位于 `src/sdk/engine.ts`，通过三因子评分（信号词命中、维度数量、内容长度）直接输出 1-10 轮。零 API 调用，毫秒级响应。

**局限性**：关键词计数是粗粒度信号，无法真正理解内容语义。例如"风险"一词在浅层建议和深层合规问题中权重相同。

---

## 方案 C：本地规则 + AI 兜底

### 核心思路

- **规则引擎覆盖明确极端**（约 70% 场景），零成本快速判定
- **中间模糊地带**（约 30% 场景）才调用一次 round_judge AI，获得语义级精度

### 判定流程

```
提取器输出 → estimateRoundCount() → 返回 { confidence, rounds, score }
                                           │
                    ┌──────────────────────┼──────────────────────┐
                    ▼                      ▼                      ▼
              confidence='high'     confidence='middle'    confidence='high'
              明显简单/明显复杂         模糊地带              明显简单/明显复杂
              直接使用 rounds      调用 round_judge API      直接使用 rounds
                                        │
                                        ▼
                                parseRoundJudgeOutput()
                                提取轮数，兜底 rounds
```

### 置信度分界

| 综合分 | 置信度 | 行为 |
|--------|--------|------|
| 0-2 | high | 直接返回 1-2 轮 |
| 3-8 | middle | 触发 AI 判定 |
| 9+ | high | 直接返回 4-10 轮 |

### 需改动的代码

1. **`estimateRoundCount`** 返回值从 `number` 改为 `{ rounds: number; confidence: 'high' | 'middle'; score: number }`
2. **`startReview`** 恢复 `roundJudgeConfig` 变量，仅在 `confidence === 'middle'` 时调用 round_judge API
3. **`parseRoundJudgeOutput`** 恢复用于从 AI 输出中提取轮数
4. **`allSteps`** 初始计算需考虑可能多一步 round_judge 调用

### 收益

| 指标 | 纯本地（当前） | 方案 C |
|------|---------------|--------|
| 精度 | 中（关键词级） | 高（语义级兜底） |
| 平均延迟 | 0ms | ~0.7s（70% 场景 0ms + 30% 场景 2-3s） |
| 平均成本 | 0 | ~0.003 元/次 |
| 代码复杂度 | 低 | 中 |
